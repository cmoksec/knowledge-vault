# AppSec

Created: June 2, 2025 3:50 PM

Application security, комплекс мер по защите информационных систем, включающий:

1. Secure Design (Risk modeling)
2. Secure Development (SSDLC)
3. Testing & Vulnerability assessment (Pentest)
4. Secure Deployment (DevSecOps)
5. Incident response (SOC)
6. Secure Supply chain
7. User & Data protection
8. Compliance & regulations

Same-Origin — тройка: протокол, домен, порт

Same-Site: — два URL считаются *same-site*, если у них **совпадает домен верхнего уровня (TLD) + часть перед ним** (eTLD+1). eTLD это, например, **co.za**

## Топ уязвимостей

1. **SQL Injection (SQLi)** - Внедрение злонамеренных SQL-команд для манипуляции или кражи данных из базы данных.
2. **Cross-Site Scripting (XSS)** - Внедрение вредоносных скриптов в веб-страницы, выполняемых в браузере жертвы.
3. **Cross-Site Request Forgery (CSRF)** - Принуждение аутентифицированного пользователя к выполнению нежелательных действий на веб-сайте.
4. **Server-Side Request Forgery (SSRF)** - Принуждение сервера к выполнению произвольных HTTP-запросов к внутренним или внешним ресурсам.
5. **XML External Entity (XXE)** - Эксплуатация обработки XML для чтения локальных файлов или выполнения удалённых запросов через внешние сущности.
6. **Remote Code Execution (RCE)** - Возможность выполнения произвольного кода на целевом сервере.
7. **Local File Inclusion (LFI)** - Чтение локальных файлов на сервере через уязвимые параметры приложения.
8. **Remote File Inclusion (RFI)** - Включение и выполнение удалённых файлов на сервере через уязвимые скрипты.
9. **Insecure Direct Object References (IDOR)** - Несанкционированный доступ к объектам (файлам, записям) путём манипуляции идентификаторами.
10. **Security Misconfiguration** - Эксплуатация некорректных настроек сервера, СУБД или приложения (например, дефолтные учетные данные).
11. **Sensitive Data Exposure** - Недостаточная защита конфиденциальных данных (пароли, токены, платежные реквизиты).
12. **Broken Authentication** - Обход механизмов аутентификации через уязвимости в логине, сессиях или управлении паролями.
13. **Broken Access Control** - Нарушение контроля доступа, позволяющее пользователям действовать вне своих полномочий.
14. **Prototype Pollution** - Модификация прототипа объекта JavaScript для изменения логики приложения или выполнения кода.
15. **HTTP Request Smuggling** - Искажение обработки HTTP-запросов между серверами (frontend/backend) для обхода защиты.
16. **Race Conditions** - Эксплуатация временных окон между операциями (например, параллельные балансовые транзакции).
17. **Web Cache Poisoning** - Подмена кешируемого контента для распространения вредоносных данных пользователям.
18. **Web Cache Deception** - Принуждение сервера к кешированию конфиденциальных данных для несанкционированного доступа.
19. **Server-Side Template Injection (SSTI)** - Внедрение произвольного кода в шаблоны сервера (Jinja2, Twig и др.).
20. **Command Injection** - Выполнение произвольных системных команд через уязвимые параметры приложения.
21. **Path Traversal** - Доступ к файлам вне целевой директории путём манипуляции путями (например, `../../etc/passwd`).
22. **Open Redirect** - Перенаправление пользователей на фишинговые сайты через подконтрольные URL.
23. **Clickjacking** - Обманное наложение интерфейса для выполнения действий без ведома пользователя.
24. **Insecure Deserialization** - Эксплуатация десериализации ненадёжных данных для выполнения кода или DoS.
25. **Denial of Service (DoS/DDoS)** - Нарушение доступности ресурса путём исчерпания его ресурсов.
26. **OAuth Misconfiguration** - Обход аутентификации через ошибки в реализации OAuth/OpenID Connect.
27. **JWT Vulnerabilities** - Подделка токенов (например, алгоритм `none`, слабые секреты, отсутствие валидации).
28. **HTTP Host Header Attacks** - Подмена заголовка Host для обхода виртуализации, кеширования или парсинга.
29. **CRLF Injection** - Внедрение управляющих символов (Carriage Return/Line Feed) для подмены HTTP-ответов.
30. **Content Spoofing** - Подмена контента приложения без внедрения кода (через HTML-инъекции или URL).
31. **DOM Clobbering** - Манипуляция DOM-элементами для изменения поведения JavaScript.
32. **CORS Misconfiguration** - Неправильная настройка Cross-Origin Resource Sharing, разрешающая несанкционированные запросы.
33. **WebSocket Hijacking** - Эксплуатация уязвимостей в WebSocket для несанкционированного доступа к данным.
34. **Form Jacking** - Кража данных платежных форм через внедрение вредоносного JavaScript.
35. **File Upload Bypass** - Обход ограничений загрузки файлов для размещения шелл-кода (например, двойные расширения).
36. **API Abuse** - Эксплуатация недостатков API: отсутствие лимитов, инъекции, неправильные проверки прав.
37. **LDAP Injection** - Внедрение команд в LDAP-запросы для обхода аутентификации или получения данных.
38. **XPath Injection** - Манипуляция XML-путями для несанкционированного доступа к данным.
39. **Session Fixation** - Навязывание пользователю известного злоумышленнику идентификатора сессии.
40. **Padding Oracle Attacks** - Расшифровка данных через анализ ответов приложения на ошибочный padding (например, в CBC).
